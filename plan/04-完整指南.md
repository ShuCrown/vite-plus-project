## 故障排查

### 问题 1: `command not found: vp`

**症状**：执行 `vp` 命令时报错

**解决方案**：

```bash
npm install -g vite-plus
# 或更新
vp upgrade
```

### 问题 2: `wrangler: command not found`

**症状**：运行 `vp exec wrangler ...` 时报错

**解决方案**：

```bash
cd apps/api
vp install
```

### 问题 3: `Unauthorized` 错误

**症状**：Cloudflare 认证失败

**解决方案**：

```bash
vp exec wrangler logout
vp exec wrangler login
```

### 问题 4: API 无法访问

**症状**：无法连接到 http://localhost:8787

**原因和解决**：

1. **API 服务器未运行**

   ```bash
   cd apps/api && vp exec wrangler dev
   ```

2. **端口 8787 被占用**
   ```bash
   lsof -i :8787
   ```

### 问题 5: 前端无法代理 API

**症状**：访问 `/api/health` 返回 404

**原因和解决**：

1. **vite.config.ts 配置不对**
   检查 `apps/web/vite.config.ts` 是否有：

   ```typescript
   server: {
     proxy: {
       '/api': {
         target: 'http://localhost:8787',
         changeOrigin: true,
       }
     }
   }
   ```

2. **前端服务器未重启**
   停止并重新启动 `vp dev`

### 问题 6: 数据库迁移失败

**症状**：`Error: Unauthorized`

**原因和解决**：

1. **database_id 不对**

   ```bash
   # 检查 wrangler.toml 中的 database_id
   cat apps/api/wrangler.toml | grep database_id
   ```

2. **未登录 Cloudflare**

   ```bash
   vp exec wrangler login
   ```

3. **数据库还未创建**
   ```bash
   cd apps/api
   vp exec wrangler d1 create my-app-db
   ```

## 部署指南

### 部署前检查清单

- [ ] 已运行 `vp check` 通过类型检查
- [ ] 已运行 `vp test` 通过测试
- [ ] 已运行 `vp build` 构建成功
- [ ] 已验证本地功能正常
- [ ] database_id 已正确填入 wrangler.toml
- [ ] 已登录 Cloudflare

### 部署步骤

#### 1. 生产数据库迁移

```bash
cd /home/yaoye/projects/agent/test/vite-plus-project/apps/api

# 执行生产迁移
vp exec wrangler d1 migrations apply my-app-db

# 验证（可选）
vp exec wrangler d1 execute my-app-db --command "SELECT COUNT(*) FROM users"
```

#### 2. 部署 API Worker

```bash
cd apps/api

# 部署
vp exec wrangler deploy

# 预期输出：
# ✓ Uploaded vite-plus-project-api
# ✓ Published vite-plus-project-api
```

**获取 API URL**：

```bash
# 从输出中获取，通常是：
# https://vite-plus-project-api.<account>.workers.dev
```

#### 3. 构建前端

```bash
cd apps/web

# 构建
vp build

# 预期：生成 dist/ 目录
```

#### 4. 部署前端到 Cloudflare Pages

```bash
cd apps/web

# 部署
vp exec wrangler pages deploy dist --project-name vite-plus-project

# 预期输出：
# ✓ Deployed vite-plus-project
```

**获取前端 URL**：

```bash
# 通常是：
# https://vite-plus-project.pages.dev
```

#### 5. 配置生产环境变量

在 Cloudflare Pages 设置中：

1. 项目 → 设置 → 环境变量
2. 添加变量：
   ```
   VITE_API_URL=https://vite-plus-project-api.<account>.workers.dev
   ```

#### 6. 更新前端代码（可选）

如果前端需要根据环境使用不同的 API URL：

```typescript
const API_URL = import.meta.env.VITE_API_URL || "http://localhost:8787";

// 使用
const response = await fetch(`${API_URL}/api/users`);
```

## 最佳实践

1. **总是使用 vp 命令**，不要直接用 npm/pnpm
2. **定期运行 vp check** 确保代码质量
3. **数据库迁移前备份**（生产环境）
4. **在分阶段部署**：先 staging 再 production
5. **使用环境变量管理敏感信息**
6. **SQL 使用参数化查询**（已在代码中实现）
7. **定期查看 Cloudflare Analytics**
