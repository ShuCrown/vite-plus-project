## 💻 常用命令速查

### 开发命令

```bash
# 安装所有依赖
vp install

# 启动前端开发（含 API 代理）
cd apps/web && vp dev

# 启动 API 开发
cd apps/api && vp exec wrangler dev

# 类型检查 + lint + format 检查
vp check

# 运行测试
vp test

# 代码格式化
vp fmt

# 代码 lint
vp lint
```

### 构建命令

```bash
# 构建前端
cd apps/web && vp build

# 本地预览构建结果
cd apps/web && vp preview

# API 不需要单独构建（wrangler 自动处理）
```

### 数据库命令

```bash
# 创建 D1 数据库
vp exec wrangler d1 create my-app-db

# 本地迁移数据库
cd apps/api && vp exec wrangler d1 migrations apply my-app-db --local

# 生产迁移数据库
cd apps/api && vp exec wrangler d1 migrations apply my-app-db

# 查询本地数据库
cd apps/api && vp exec wrangler d1 execute my-app-db --local --command "SELECT * FROM users"

# 查询生产数据库
cd apps/api && vp exec wrangler d1 execute my-app-db --command "SELECT * FROM users"
```

### 部署命令

```bash
# 部署 API Worker 到生产
cd apps/api && vp exec wrangler deploy

# 部署前端到 Cloudflare Pages
cd apps/web && vp exec wrangler pages deploy dist --project-name vite-plus-project

# 查看部署历史
cd apps/api && vp exec wrangler deployments list
```

## 🛠️ 常见问题速解

| 问题                    | 症状                             | 解决方案                                                                                    |
| ----------------------- | -------------------------------- | ------------------------------------------------------------------------------------------- |
| **vp 命令不存在**       | `command not found: vp`          | `npm install -g vite-plus` 或 `vp upgrade`                                                  |
| **wrangler 命令不存在** | 运行 API 脚本时报错              | `cd apps/api && vp install`                                                                 |
| **Cloudflare 认证失败** | `Unauthorized` 或 `401`          | `vp exec wrangler logout && vp exec wrangler login`                                         |
| **API 无法访问**        | 无法连接到 http://localhost:8787 | 1. 确认 API 服务器在运行<br>2. 检查 8787 端口是否被占用：`lsof -i :8787`                    |
| **前端无法代理 API**    | 请求 `/api/*` 返回 404           | 检查 `apps/web/vite.config.ts` 中的 proxy 配置                                              |
| **数据库迁移失败**      | `Error: ...`                     | 1. 确保 `database_id` 正确<br>2. 确保已登录 Cloudflare<br>3. 查看 04-完整指南.md 的故障排查 |
| **前端构建失败**        | 构建时报错                       | 运行 `vp check` 查看详细错误                                                                |
| **npm/pnpm 依赖冲突**   | 安装时报错                       | 使用 `vp install` 而不是 `npm install` 或 `pnpm install`                                    |

## ✅ 常用工作流

### 工作流 1：本地开发（日常）

```bash
# 终端 1
cd apps/api && vp exec wrangler dev

# 终端 2
cd apps/web && vp dev

# 浏览器访问 http://localhost:5173
# API 自动代理到 http://localhost:8787
```

### 工作流 2：添加 API 路由

1. 编辑 `apps/api/src/index.ts`
2. 添加新的 Hono 路由（热重载自动更新）
3. 在前端访问新路由

### 工作流 3：修改数据库 Schema

1. 创建新迁移文件：`apps/api/migrations/0002_xxx.sql`
2. 本地测试：`vp exec wrangler d1 migrations apply my-app-db --local`
3. 生产部署前：`vp exec wrangler d1 migrations apply my-app-db`

### 工作流 4：部署到生产

```bash
# 1. 类型检查和测试
vp check && vp test

# 2. 构建前端
cd apps/web && vp build

# 3. 生产 DB 迁移
cd apps/api && vp exec wrangler d1 migrations apply my-app-db

# 4. 部署 API
vp exec wrangler deploy

# 5. 部署前端
cd apps/web && vp exec wrangler pages deploy dist
```

## 🔐 安全检查清单

### 部署前必须检查

- [ ] `database_id` 已正确填入 `wrangler.toml`
- [ ] 已运行 `vp check` 和 `vp test`
- [ ] 已运行 `vp build` 确保构建成功
- [ ] 未提交敏感信息（`.dev.vars` 在 .gitignore 中）
- [ ] SQL 使用参数化查询（已在代码中实现）
- [ ] CORS 配置不使用通配符 `*`（生产环境）

### 生产环境配置

在 Cloudflare Pages 设置中添加环境变量：

```
VITE_API_URL=https://你的-api-worker-url.workers.dev
```

前端代码中使用：

```typescript
const API_URL = import.meta.env.VITE_API_URL || "http://localhost:8787";
```

## 🚨 重要提醒

### ⚠️ 必须做

1. **替换 database_id**

   ```toml
   # wrangler.toml
   database_id = "真实的-database-id"
   ```

2. **使用 vp 命令**

   ```bash
   ✅ vp install
   ✅ vp dev
   ✅ vp test
   ❌ npm install
   ❌ pnpm dev
   ❌ npm test
   ```

3. **运行 2 个终端**
   - 终端 1：API (`wrangler dev`)
   - 终端 2：前端 (`vp dev`)

### ⚠️ 不要做

- ❌ 修改 wrangler.toml 中除 database_id 外的配置
- ❌ 删除或修改迁移文件（除非你知道后果）
- ❌ 在生产前跳过 `vp check` 和 `vp test`
- ❌ 不登录就直接部署
