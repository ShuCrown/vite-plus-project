## 🚨 重要提醒

### ⚠️ 必须做的

1. **database_id 必须更新**

   ```toml
   # wrangler.toml 中这一行必须替换
   database_id = "你的-真实-database-id"
   ```

   不做这一步，部署会失败！

2. **使用 `vp` 命令，不用 npm/pnpm 直接运行**

   ```bash
   ❌ npm install  →  ✅ vp install
   ❌ pnpm dev     →  ✅ vp dev
   ❌ npm test     →  ✅ vp test
   ```

3. **同时运行 2 个终端**
   - 终端 1：API server (`wrangler dev`)
   - 终端 2：Front-end (`vp dev`)

### ⚠️ 不要做的

- ❌ 直接编辑 wrangler.toml 的其他配置（除了 database_id）
- ❌ 删除或修改迁移文件
- ❌ 在生产前跳过 `vp check` 和 `vp test`
- ❌ 不登录就直接部署

## 🔧 常见问题速解

| 问题                          | 解决方案                                                |
| ----------------------------- | ------------------------------------------------------- |
| `vp: command not found`       | `npm install -g vite-plus` 或 `vp upgrade`              |
| `wrangler: command not found` | `cd apps/api && vp install`                             |
| `Unauthorized` 错误           | `vp exec wrangler logout && vp exec wrangler login`     |
| API 无法访问                  | 检查 API 服务器是否运行，检查 vite.config.ts proxy 配置 |
| 前端构建错误                  | 运行 `vp check` 检查类型错误                            |
| 数据库迁移失败                | 确保 database_id 正确，确保已登录                       |

**更多问题？** 查看 **04-完整指南.md** 的"故障排查"部分
