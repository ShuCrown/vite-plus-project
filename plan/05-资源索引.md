## 💡 最佳实践

### 开发实践

1. **本地开发时总是运行 2 个终端**

   ```bash
   # Terminal 1
   cd apps/api && vp exec wrangler dev

   # Terminal 2
   cd apps/web && vp dev
   ```

2. **定期检查代码质量**

   ```bash
   vp check  # 每次提交前
   vp test   # 至少每周一次
   ```

3. **使用环境变量管理配置**
   - 本地：`.dev.vars`
   - 生产：Cloudflare 设置界面

4. **数据库迁移前备份**
   ```bash
   # 重要数据一定要备份！
   ```

### 安全实践

1. **SQL 始终使用参数化查询**

   ```typescript
   // ✅ 正确
   await c.env.DB.prepare("SELECT * FROM users WHERE id = ?").bind(id).first();

   // ❌ 错误
   await c.env.DB.prepare(`SELECT * FROM users WHERE id = ${id}`).first();
   ```

2. **敏感信息不提交到 Git**
   - `.dev.vars` 在 .gitignore 中
   - API keys 存储在环境变量中

3. **CORS 配置明确**

   ```typescript
   // ❌ 生产不要用通配符
   cors({ origin: "*" });

   // ✅ 指定具体来源
   cors({ origin: "https://yourdomain.com" });
   ```

4. **验证用户输入**
   ```typescript
   // 检查必填字段
   if (!name || !email) {
     return c.json({ error: "Missing fields" }, 400);
   }
   ```

### 性能实践

1. **数据库查询添加索引**

   ```sql
   CREATE INDEX idx_users_email ON users(email);
   ```

2. **使用数据库连接池**
   - Cloudflare D1 自动处理

3. **实现缓存**

   ```typescript
   // 使用 Cloudflare KV 缓存热点数据
   const cached = await c.env.CACHE.get(`users:${id}`);
   ```

4. **代码分割**
   - Vite+ 自动处理
   - React.lazy 路由级分割

## 📞 常见问题快速索引

### 代码相关

- Q: 如何添加新的 API 路由？
  - A: 编辑 `apps/api/src/index.ts`，添加新的 `app.get/post/put` 等

- Q: 如何修改数据库？
  - A: 创建新迁移文件，运行迁移

- Q: 前端怎么调用 API？
  - A: 使用 `fetch` 或 axios，URL 为 `/api/*`（自动代理）

### 部署相关

- Q: 如何更新已部署的版本？
  - A: 修改代码后，再次运行 `wrangler deploy` 和 `pages deploy`

- Q: 生产环境找不到 API 怎么办？
  - A: 检查 `VITE_API_URL` 环境变量是否配置正确

- Q: 如何回滚部署？
  - A: 在 Cloudflare Dashboard 中查看部署历史，使用 rollback 功能

### 性能相关

- Q: API 响应慢怎么办？
  - A: 1. 检查数据库查询，2. 添加索引，3. 使用缓存

- Q: 前端加载慢怎么办？
  - A: 1. 代码分割，2. 懒加载，3. CDN 缓存
